# LIMEN Project Instructions

- Build mobile-first.
- All user-facing copy must be written in Spanish unless explicitly requested otherwise.
- Keep the product simple and understandable.
- Avoid premature abstraction.
- Do not add dependencies without a concrete need.
- Do not add Supabase or any backend until explicitly requested.
- Do not add authentication, payments, dashboards, or a self-service editor until explicitly requested.
- Do not invent LIMEN’s final logo, color palette, typography, illustrations, or brand identity.
- The permanent product UI direction is inspired by modern iOS principles: clarity, generous spacing, strong hierarchy, refined grouped surfaces, comfortable touch targets, subtle depth, excellent accessibility, and mobile-first interaction.
- Do not copy Apple interfaces literally or use Apple proprietary assets.
- Avoid generic SaaS styling, excessive glassmorphism, dense layouts, decorative clutter, heavy shadows, and excessive pills.
- Prefer straightforward, readable TypeScript and React.
- Use semantic HTML.
- Preserve accessibility.
- Run lint, typecheck, and build before completing implementation tasks.
- Fix validation errors rather than suppressing them.
- Report all changed files, assumptions, and validation results.
